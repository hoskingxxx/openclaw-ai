## 步骤 2：配置 OpenClaw 使用 DeepSeek

### 2.1 找到 OpenClaw 配置文件

OpenClaw 的配置文件通常在：

```bash
# macOS/Linux
~/.openclaw/config.yaml

# 或者在项目根目录
./openclaw.config.yaml
```

### 2.2 修改模型配置

用你喜欢的编辑器打开配置文件，找到 `agent.model` 部分：

```yaml
# 原始配置（使用 Claude）
agent:
  model: "anthropic/claude-opus-4-5"
  api_key: "your-claude-api-key"

# 新配置（使用 DeepSeek）
agent:
  model: "deepseek/deepseek-r1"
  api_key: "your-deepseek-api-key"
  base_url: "https://api.deepseek.com/v1"  # 可选，指定 API 端点
```

### 2.3 环境变量方式（推荐）

更安全的方式是使用环境变量：

```bash
# 创建 .env 文件
echo "DEEPSEEK_API_KEY=your-api-key-here" >> .env

# 或者在命令行直接设置
export DEEPSEEK_API_KEY="your-api-key-here"
```

然后在配置文件中引用：

```yaml
agent:
  model: "deepseek/deepseek-r1"
  api_key: "${DEEPSEEK_API_KEY}"
```

## 高级配置：使用本地 DeepSeek（完全免费）

如果你想完全零成本运行，可以在本地部署 DeepSeek 模型：

### 前置要求

- Mac M1/M2/M3 或 NVIDIA 显卡
- 至少 16GB 内存

### 安装 Ollama + DeepSeek

```bash
# 1. 安装 Ollama
brew install ollama

# 2. 下载 DeepSeek R1 模型（约 4GB）
ollama pull deepseek-r1:7b

# 3. 启动 Ollama 服务
ollama serve

# 4. 配置 OpenClaw 使用本地模型
```

```yaml
agent:
  model: "ollama/deepseek-r1:7b"
  base_url: "http://localhost:11434/v1"
  api_key: "ollama"  # Ollama 不需要真实的 key
```

**优点**：完全免费、数据不联网、隐私最高

**缺点**：需要较好的硬件配置

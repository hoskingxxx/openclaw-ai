## 什么是 OpenClaw？

### 核心理念

从「你问我答」到「你下目标，我来执行」

AI 不再是工具，而是**数字员工**

### 一句话定义

> OpenClaw 是一个开源的执行型 AI Agent，能真正替你干活——读文件、跑命令、改代码、自动化流程。

### 它不是什么

- ❌ 不是只能回答问题的聊天机器人
- ❌ 不是依赖云端服务的黑盒产品
- ❌ 不是需要打开专门界面的工具

### 它是什么

- ✅ 可以根据你的指令去执行任务
- ✅ 可以部署在你自己的电脑或服务器上
- ✅ 能通过你常用的聊天工具接收指令
- ✅ 能访问本地文件、管理邮件日程、运行脚本、整理数据

## 核心特点

| 特点 | ChatGPT | OpenClaw |
|------|---------|----------|
| **角色** | 顾问 | 执行者 |
| **工作方式** | 你问 → 它答 | 你给目标 → 它拆解 → 执行 → 汇报 |
| **能力** | 回答问题、给建议 | 读文件、跑命令、改代码、执行流程 |
| **部署** | 云端 | 本地/服务器 |
| **数据** | 需要上传 | 私有可控 |

### 六大核心特点

**1️⃣ 执行型 AI，不是聊天型 AI**

你给目标 → 它自己拆步骤 → 调工具 → 执行 → 汇报结果

它是"做事的人"，不是"出主意的人"

**2️⃣ 真正的行动能力**

- 读写本地文件
- 执行 shell 命令
- 调用 LLM（GPT/Claude/本地模型）
- 串联多个工具完成一个目标

**3️⃣ 目标驱动，不是 Prompt 驱动**

ChatGPT 思维：`帮我优化一下这个项目`

OpenClaw 思维：
```
目标：修复当前报错
范围：仅 src/ 目录
约束：不新增依赖
输出：patch + 原因说明
```

**4️⃣ 长期工作，有记忆和上下文**

- 持续跟踪一个项目
- 记住中间状态
- 在失败后重试
- 按你的规则停止/回滚

**5️⃣ 本地/私有可控**

- 你的代码/文件不需要上传到第三方
- 你能精细控制它的权限
- 它能访问真实环境（而不是沙盒）

**6️⃣ 天然支持 Agent 架构**

- 单 Agent：一个 AI 员工
- 多 Agent：A 负责分析、B 负责执行、C 负责校验

## 6 大实用玩法

### 玩法 1️⃣：AI 程序员（最火）

**适合谁：** 独立开发者、工程师、在维护旧项目的人

**能做什么：**
- 读整个代码仓库
- 定位 bug
- 修改代码
- 输出 patch/diff

**示例指令：**
```
目标：修复当前项目的报错
范围：src/ 目录
约束：不新增依赖
输出：diff + 原因说明
```

---

### 玩法 2️⃣：AI 运维/SRE

**适合谁：** 运维、技术负责人、有服务器的人

**能做什么：**
- 查日志
- 跑命令
- 重启服务
- 自动排查故障

**示例指令：**
```
检查最近 24 小时服务异常
分析日志
给出最小修复方案
不要执行破坏性命令
```

---

### 玩法 3️⃣：自动化数据处理

**适合谁：** 做数据/运营/SEO、经常处理 CSV/Excel 的人

**能做什么：**
- 清洗数据
- 合并文件
- 统计/分析
- 输出新文件

**示例指令：**
```
读取 data/ 下所有 CSV
清洗空值
合并为一个表
输出 summary.csv
```

---

### 玩法 4️⃣：本地知识助理

**适合谁：** 有大量文档的人、不想上传资料到云端的人

**能做什么：**
- 只基于你本地资料回答问题
- 查制度/文档/项目说明
- 总结关键信息

**示例指令：**
```
仅基于 docs/ 目录
回答：我们的退款规则是什么
给出处文件路径
```

---

### 玩法 5️⃣：SEO/内容批量生产

**适合谁：** 站长、内容创业者、做 niche site 的人

**能做什么：**
- 扫描目录
- 批量生成内容
- 改写/优化文案
- 输出结构化页面

**示例指令：**
```
读取 keywords.txt
为每个关键词生成 SEO 页面草稿
输出为 markdown 文件
```

---

### 玩法 6️⃣：多 Agent 协作（进阶）

- Agent A：拆解任务

## 常见问题

### Q1：OpenClaw 和 ChatGPT 有什么本质区别？

**一句话：ChatGPT 负责"想"，OpenClaw 负责"干"。**

| ChatGPT | OpenClaw |
|---------|----------|
| 回答问题、给建议 | 读文件、跑命令、改代码、执行流程 |

---

### Q2：我需要会编程才能用吗？

**基础使用：** 不需要写代码，但要懂基本逻辑

**深度使用：** 懂一点命令行/项目结构会非常加分

> 它不是"零门槛"，而是"低门槛高上限"。

---

### Q3：能在 Windows/Mac/Linux 上用吗？

- ✅ Mac：最友好
- ✅ Linux/服务器：生产环境首选
- ⚠️ Windows：通常通过 WSL

---

### Q4：OpenClaw 安全吗？

**技术上中立，责任在使用者。**

风险来自：
- 文件权限
- 系统命令
- API Key 管理

本质上，它和一个"有系统权限的员工"一样安全。

---

### Q5：OpenClaw 免费吗？

**框架本身：** 开源免费

**成本主要在：** 模型 API/服务器

---

### Q6：OpenClaw 自己有大模型吗？

**没有。**

它是一个"调度器"，需要你自己接：
- OpenAI
- Claude
- 或本地模型

> 它不卖模型，只负责让模型"动起来"。

---

### Q7：会"自己乱来"吗？

如果你权限给太大，会。

OpenClaw 的能力 ≈ 你给的权限

**正确姿势：**
- 默认只读
- 明确可写目录
- 禁止危险命令

---

### Q8：适合在生产环境用吗？

适合，但前提是：
- 你清楚它能做什么
- 有权限隔离
- 有日志/回滚机制

新手不建议一上来就上生产。

## 新手安全速查

### ✅ 推荐做法

- 默认只读权限
- 明确可写目录
- 打开日志
- 小任务先跑通

### ❌ 避免做法

- 不给 root/全盘权限
- 不要给模糊目标
- 不要忽视日志
